Week 5 - Export / Import

Import and export

Import and export are two ways to include a module in another module. The import statement can be used to import functions, objects, or primitive values that have been exported from an external module.

The export statement is used when creating JavaScript modules to export functions, objects, or primitive values from the module so they can be used by other files with the import statement.
There are two types of exports: export default and export. The export default is used to export a single value or a function from a module. The export statement is used to export multiple values or functions from a module.

In HTML you can use the <link> tag to import a CSS file and the <script> tag to import a JavaScript file. The script tag can also be used to import a module. <script type="module" src="module.js"></script>
This tells the browser that the linked JavaScript file (module.js in this case), should be treated as a module. Using modules helps organize and modularize your JavaScript code, making it easier to manage and reuse across different parts of your application.

In Node.js you can use the require() function to import a module, which is the CommonJS way of importing modules. When importing express for example, we could write:
const express = require('express');
The require() function returns the module.exports object, which contains all the exports from the module that is being required.
We can also import the express library using the ES6 way of importing modules:
import express from 'express';
It is the recommended way to import modules in modern JavaScript.

Serving static files and security needs

The code app.use(express.static('public')); is used in Node.js applications to serve static files such as HTML, CSS and images.
app refers to an instance of the Express application.
express.static() is a built-in middleware function in Express. It serves static files and is based on the serve-static middleware module.
'public' is the directory from which to serve static assets.
So, when a request is made to your Express server for a static file, Express will look for that file in the "public" directory and serve it (if it exists).

Security

When serving static files, it is important to consider security when publishing the application. If you serve files from a directory that is accessible to the public, it is important to be careful not to serve files that contain passwords, API keys, or other sensitive data.
We can prevent it by removing sensitive files from the public directory, using .gitignore or by using environment variables to store sensitive data.

Frontend structure

The frontend structure is the way we organize the assets, pages and components in our frontend, which are the static files that are located in the public directory.
A good structure makes the code more readable and maintainable.
- The assets are the files that are used in the frontend, (images, CSS, and JS files).
- The pages are the HTML files that are served to the user.
- Components are reusable parts of the frontend, such as a header, footer, or navigation bar.

A good structure for the frontend is to have a separate directory for assets and pages. The assets directory should contain subdirectories for images, CSS, and JavaScript files. The pages directory should contain the HTML files that are served to the user.

Learning goals:

  • Can work with both types of import/export in HTML.
  • Can import/export in both ways in Node.js.
  • Understands the security need for serving static files and can do it in Express.
  • Can argue for a better way to structure the frontend (assets, pages) and how to structure HTML.
