Week 2 - Creating the first server

Scoping

Scoping in JavaScript refers to the accessibility of variables. In other words, which parts of your program can see or use it. There are three types of scopes in JavaScript: global, local, and block.

Always use 'const' as default when declaring variables.
const is constant NOT in the value, but in the assignment, meaning that it cannot be reassigned and thus also HAS to be assigned.

Use 'let' when you need to reassign a variable.

'var' is the old way of declaring variables and should be avoided.

Advanced functions

Functions can be assigned to variables, passed as arguments, and returned from other functions.

Callback functions are functions that are passed as arguments to other functions. '() => {}' - Arrow functions is a short way to write functions in JavaScript.

The 'genericActionPerformer' function takes "running" as a callback and calls it with a provided name parameter.

package.json

The package.json file is the heart of a Node.js project. It contains dependencies, metadata about the project, such as the name, version, description, and other details.

To create a package.json file, you can run npm init in the terminal.

  1. Then you will be prompted to provide various information about your project, like package name, version, description, author, and license. You can fill in these details or just press 'Enter' to accept the default values.
  2. After accepting the values for each prompt, npm will display a summary of the information you provided. Type "yes" and press 'Enter' to generate the package.json file.
  3. The package.json file will be in the project directory that contains the metadata and configuration for your Node.js project.

node_modules

The node_modules directory is where npm installs packages. It is a subdirectory of the project directory where the package.json file is located.

The node_modules directory is not committed to the version control system, and it is not deployed to the production environment.
