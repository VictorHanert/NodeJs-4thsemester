Week 6 - Server-side Rendering

Client-side rendering (CSR)

  • Definition: CSR is a method of rendering web pages where the HTML content is small, and stuff like data fetching, template rendering, and UI updates, is handled by JavaScript running in the client's web browser.
    CSR allows us to make websites entirely rendered in the browser with JavaScript. Instead of having a different HTML page per route, a client-side rendered website creates each route dynamically directly in the browser.
  • Pros:
    • Fast initial load time.
    • Fast navigation between pages.
    • Better suited for highly interactive and dynamic web applications.
    • Less complex server setup.

Server-side rendering (SSR)

  • Definition: SSR is a method of rendering web pages where the HTML content is generated on the server and sent to the client.
    SSR is the traditional method of rendering web pages, where the server sends the HTML content to the client, and the client's web browser renders the page.
  • Pros:
    • Faster load time
    • Uses less resources
    • Good for SEO.
    • Good for websites with a lot of content.
    • Especially for companys: better for solving all CORS problems.

Redirections

Client-side redirection is handled by the browser using JavaScript or HTML meta tags.
Example: window.location.href = 'new_page_url';
The redirection happens as soon as the browser receives the response from the server, and can redirect users based on certain conditions or events.

Server-side redirection is handled by the web server before the response is sent to the client, which gives good reliability and better SEO performance.
Example: res.redirect('new_page_url');

In the example above, we have a simple server that listens for a GET request to the /logIn route. We can test the endpoint by typing the path and query string in the browser: /logIn?password=SesameOpenUp
If the password is incorrect, the server sends a 400 status code and a message to the client.
And if it's correct then the server redirects the user to the /loggedIn route, and displays the message "You are now logged in!".
